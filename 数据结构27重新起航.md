```c
//用并查集判断一个图有几个连通分量（图用邻接矩阵表示）
int ComponentCount(int g[5][5]){
    //g[5][5]是二维数组表示的邻接矩阵
    int S[5]; //定义、初始化并查集
    for(int i=0;i<5;i++)
        S[i]=-1;
    //遍历各条边（无向图，遍历上三角部分即可）
    for(int i=0;i<5;i++)
        for(int j=i+1;j<5;j++)
            if(g[i][j]>0){ //结点i，j之间有边
            	int iRoot=FInd(S,i)
            	int iRoot=Find(S,j);
            	if (iRoot!=JRoot) //i,j并入同一集合
            		Union(S,iRoot,jRoot):
            }
    //统计有几个连通分量
    int count=0;
    for(int i<0;i<5;i++)
        if(S[i]<0) count++;
    return count;
}
```

**1. `malloc` 函数**

- **作用**：向操作系统申请一块**连续的堆内存空间**。
- **参数**：`5 * sizeof(int)`
  - `sizeof(int)`：计算一个 `int` 类型变量占用的字节数（通常为 4 字节）。
  - `5 * sizeof(int)`：表示申请 `5` 个连续的 `int` 类型空间（总大小为 `5*4=20` 字节）。
- **返回值**：返回分配内存的**起始地址**（类型为 `void*`，即“无类型指针”）。

## 顺序表和链表的对比

|            | 顺序表 | 链表 |
| :--------: | :----: | :--: |
|  可扩容性  |   弱   |  强  |
| 增，删操作 |   弱   |  强  |
|  查找操作  |   强   |  弱  |

- 表长难以预估、经常要增加/删除元素、强调逻辑关系 ——链表
- 表长可预估、查询(搜索)操作较多——顺序表



### 注意事项

- 需要依次比较各个分量来判断两个结构体是否相等

- 动态分配并不是链式存储，它同样属于顺序存储结构，物理结构没有变化，依旧是随机存取方式，只是分配的空间大小可以在运行时动态决定
- 链式存储结构比顺序存储结构更能方便地表示各种逻辑结构
- **顺序存储结构和链式存储结构都可以进行顺序存取**
- 链式存储存储空间可以不连续，但是存储单元一定连续
- **线性表的长度不包括头结点**

# 队列

## 队列应用

- 队列可以实现二叉树的层次遍历
- 队列可以实现缓冲区来解决外设和主机之间速度不匹配问题



![[4ce7b79418b5e033c6a44b7770cc1407_MD5.png]]

- 队列(栈)的两种存储方式
  - 入队(栈)的一端(队尾or栈顶)，指针指向当前元素，那么入的时候先移动指针，再入。
  - 入队(栈)的一端(队尾or栈顶)，指针指向当前元素的后(上)一个元素，那么入的时候先入，再移动指针
  - 两种方式都是front不变,入队(rear+1)%n,入队只动队尾指针

## 循环队列

- 用模运算，将空间在逻辑上变为环状
  - 循环队列指的是顺序存储的队列，而不是指逻辑上的循环，**如循环单链表表示的队列不能称为循环队列**

### 1.牺牲单元法（最常用）

队空条件：Q.front==Q.rear

**元素个数:(Q.rear-Q.front+MaxSize)%MaxSzie**

### 2.size变量法

**增设一个size变量**，表示元素个数，入队成功size+1，出队成功size-1。

队空时，Q.size==0；

队满时，Q.size==MAXsize

# 字符串模式匹配

## 基本概念

字符串，简称串，是由0个或多个字符所组成的有序序列，一般记为S=a~1~a~2~a~3~..a~n~

1. s是串名
2. a~i~可以是字母、数字或特殊字符

字符串和线性表的逻辑结构类似，主要区别在于**字符串的数据对象只能是字符**。按存储结构来分类，顺序存储的字符串称为**顺序串**，链式存储的字符串称为**链串**

**字符串的模式匹配**

字符串的模式匹配是指在一个字符串T中查找与给定字符串S相等的子串，给定字符串S叫做**模式串**，字符串T叫做**目标串**

在主串中找到与模式串相同的子串，并返回其所在位置。（模式串是想要搜索的某个字符串）



## 朴素模式匹配算法

暴力算法

设主串长度为n,模式串长度为m；

将主串中**所有长度为m的子串**依次与模式串对比，直到找到一个*完全匹配的子串*，或所有的子串都不匹配为止。

- 所以最多对比n-m+1个子串，每趟最多m次比较，最坏时间复杂度o(mn)
  - 每个子串对比m个字符，共n-m+1个子串，复杂度（n-m+1）*m  ，n>>m，所以为o(mn)
- ![[2a5bdc764e31943d3795db6786670a8e_MD5.png]]

若当前子串匹配失败，则**主串指针i指向下一个子串的第一个位置，模式串指针j回到模式串的第一个位置**

## KMP算法

**程序知道匹配失败前扫描过的字符的匹配信息**

- 步骤：观察字符串信息->寻找最长相等前后缀->next数组->将前缀移动到原本后缀的位置
  - next数组只和模式串有关，和主串无关：当模式串的==第i位==发生失配时，模式串的==第next[i-1]位==将重新与主串当前位置进行比较（一定要注意**数组下标和位序的区别**）
- 最坏时间复杂度O(m+n)--------求next数组时间复杂度O(m),模式匹配过程最坏时间复杂度O(n)

# 树与二叉树

- 结点的深度就是结点所在的层次，结点的层次从树根开始定义
  - 从上往下数

- 结点的高度就是以该结点为根的子树的高度

  - 从下往上数

- 结点的度就是**有几个分支**（有几个孩子结点）

- 树的度就是**结点的度最大的那个**

- 深度优先搜索：==优先访问孩子节点==

- 广度优先搜索：==优先访问兄弟节点==

  |                     1.结点数=度数之和+1                      |
  | :----------------------------------------------------------: |
  |                **2.度为m，第i层至多有m^i-1^**                |
  |     **3.高度为h的m叉树，至多有==(m^h^-1)/(m-1)==个结点**     |
  | **4.度为m、具有n个结点的树的最小高度h为==log~m~(n(m-1)+1)==** ==向上取整== |
  |        5.度为m、具有n个结点的树的最大高度h为==n-m+1==        |
  |                    6.==n个结点，n-1条边==                    |


## 二叉树

| 普通二叉树                                                   | 完全二叉树                                                   |
| ------------------------------------------------------------ | :----------------------------------------------------------- |
| 非空二叉树的第k层最多有==2^k-1^==个节点（k>=1）              | 完全二叉树的高度为log~2~（n+1）上取整或log~2~n下取整+1（由==2^h-1^-1<n<=2^h^-1==或==2^h-1^<=n<2^h^==得来） |
| 高度为h的二叉树至多有==2^h^-1==个节点(k>=1)                  | 编号为i的节点所在的层次为==log~2~i下取整+1==                 |
| n~0~=（2-1）n~2~+1（后面的最佳归并树同理n~0~=（m-1）n~m~+1） |                                                              |
| n个节点，n+1个空指针（最好验证一下）                         |                                                              |

> [!important] 重点
> **叶结点个数**n~0~=（m-1）n~m~+1的推导过程：
>
> 1. n=n~0~+n~m~
> 2. n=e+1   顶点数=边数+1
> 3. e=m*n~m~
> 4. m*n~m~+1=n~0~+n~m~，即n~0~=（m-1）n~m~+1

### 二叉树错题

- 交换一颗二叉树左右子树之后，中序遍历的升序序列会变成降序序列
- 前不能前：前驱线索二叉树没有**有效算法**能够通过*指针*找到**前驱**
- 后不能后：后序线索二叉树没有**有效算法**能够通过*指针*找到**后继**
- 后序线索树的遍历需要使用**栈**
- 某二叉树的先序序列和后序序列正好相反，则该二叉树只有*根节点*，或者根节点只有**左子树**或者**右子树**
- 二叉树中序遍历的最后一个结点一定是从根节点开始沿右子女指针链走到底的结点
- n个结点的树中有n-1条边，所以知道森林的**总结点和总边数**就能求出树有**多少颗**，x=n-e

}

# 图

## 图的基本概念

- 若一个图有n个顶点，且有大于n-1条边，则此图一定有环
- 图的顶点集一定是**非空集**，边集可以是**空集**

> [!important] 重点
> **连通**：两个顶点有路径存在，则这两个顶点连通
>
> **连通图**：任意两个顶点都连通(**无向图中讨论连通，有相同讨论强连通**)
>
> **强连通图**：任意两个顶点之间互相都有路径，则这两个顶点强连通
>
> **连通分量**：无向图中的极大连通子图（子图必须连通，而且尽可能地包含更多的顶点和边）
>
> **强连通分量**：有向图中的极大连通子图(有环路)
>
> **完全图**：任意两个顶点都存在边（**注意路径和边的区分**）
>
> **生成树**：连通图的生成树是包含图中**全部顶点**的一个**极小连通子图**。若顶点为n，则他的生成树含有n-1条边

> [!important] 重点
> **极小连通子图**：
>
> - 极小连通子图实际上就是该图的一个生成树。
> - 它拥有 n−1 条边（其中 n 是顶点的数量），这是保证图**连通且不形成环路**所需的最少边数。
>
> **极大连通子图**：
>
> 1.**极大**，不被任何别的子图所包含；2.**连通**；3.**子图**

### 广度优先

是类似于层序遍历，是分层的查找。

它不像深度优先那样有往回退的情况，所以它是一个非递归的算法

- **广度优先的存储主要采用邻接矩阵和邻接表**
  
  - 邻接矩阵，查找每个顶点的邻接点所需的时间为O(V)，因为共有v个顶点，总时间复杂度为O(v^2^)
  - 邻接表，每个顶点均需要搜索一次，时间复杂度为O(V),在搜索每个顶点的邻接点时，每条边至少访问一次，时间复杂度为O(E),总的时间复杂度为O(V+E)
  
  

#### 广度优先生成树

由连通图广度优先遍历而来。

- 同一个图的**邻接矩阵**存储表示是唯一的，所以其广度优先生成树也是**唯一**的
- 同一个图的**邻接表**表示是不唯一的，所以其广度优先生成树也**不唯一**



**广度优先森林**由非连通图的广度优先遍历而来



### 深度优先

类似于树的先序遍历

DFS算法是一个递归算法，因为它需要回退。

**它的时间复杂度和空间复杂度BFS一样**，但是不同的是DFS的空间复杂度主要**来自于它的递归工作栈**

- DFS的规则：当V~i~入栈后，必须先遍历完它的后继结点之后Vi才会出栈 **P247.41**

#### 深度优先生成树   

对连通图调用DFS才会产生深度优先生成树，否则产生的是深度优先生成森林。



- **对于无向图进行DFS或BFS遍历时，调用BFS或DFS函数的次数就是连通分量数**

- **对于连通图，只需要调用一次BFS或DFS**
- **对于有向图进行DFS或BFS遍历时，若起始顶点到所有顶点都有路径，则只需要调用一次DFS/BFS函数**
  - *对于强连通图，任意顶点都只需要调用一次DFS/BFS函数*

### 最小生成树

> [!important] 重点
> ==最小生成树中某顶点到其余各顶点的路径不一定具有最短路径的性质==，<u>你要说普利姆和克鲁斯卡尔可以求最短路径还真不一定对，举一个三个节点的反例就行</u>

一个连通图的生成树包含图的所有顶点，并且只含尽可能少的边。对于生成树来说，若砍去一条边，则会使生成树变成非连通图；若增加一条边，则会在图中形成一条回路

*对于一个带权连通图G，生成树不同，每棵树的总权也可能不同。权值最小的那颗树称为G的最小生成树。*

- 无向连通图各边权值都不同，则最小生成树唯一
- 最小生成树的边数等于顶点数-1

**普里姆算法Prim**

 ![[3b2b6f23c8c109b77b815475ceceebc5_MD5.webp]]

- 初始时从图中*任取*一顶点加入树T，每次将代价最小的新顶点纳入生成树，直到所有顶点都纳入为止
  - 适合边稠密图，O(V^2^)
    - 算法思想：每一轮处理*循环遍历所有个结点，找到lowCast、最低的，且还没有加入树的顶点；再次循环遍历，更新还没加入的各个顶点的lowCast值*，**这两次遍历使得每一轮为O(2n)**,从V~0~开始需要n-1轮处理

**克鲁斯卡尔算法Kruskal**

> [!important] 重点
> 边表排序+遍历+判环

![[21dcedb6189441024947cde7abad2796_MD5.png]]

- 每次选择权值最小的边，使这个边两头相通(原本已经连通的就不选)，直到所有结点连通
  - 适用于边稀疏图 O(E*log~2~E)
    - 先将各条边按权值排序，*第i轮：检查第i条边的两个顶点是否连通(是否属于同一个集合)*共执行e轮，**每轮判断两个顶点是否属于同一集合**，需要O(log~2~e)

## 拓扑排序

> [!important] 重点
> ==拓扑序列是按照点的先后顺序排列的，也就是说入度为0的点一定是排在前面的，我们直接对一个图BFS一遍，BFS过程中更新每个点的入度，如果一个点的入度为0，那么就将其加入拓扑序，并且删除其与后继结点的所有边。==

- AOV网：AOV网是一个有向无环图(DAG),有向边<V~i~,V~j~>表示活动Vi必须先于V~j~进行
- 拓扑排序：在图论中，由一个有向无环图的顶点组成的序列
  - 从AOV网中选择一个没有前驱的顶点输出
  - 从网中删除该顶点和所有以它为起点的有向边
  - 重复以上两条操作直至当前的AOV网为空(或当前网中不存在无前驱的顶点，这种情况说明有向图中必有环)
- 逆拓扑排序：拓扑排序中的操作对象改变了，*前驱*->*后继*，*以它为起点*->*以它为终点*
  - 考点：用DFS实现拓扑排序和逆拓扑排序
- 注意事项：**拓扑排序，逆拓扑排序序列可能不唯一**，**若图中有环，则不存在拓扑排序序列/逆拓扑排序序列**
- 强连通图和拓扑排序互斥

- 散列表，又叫**哈希表，可以根据数据元素的关键字计算它在散列表中的存储地址**
- 散列函数，又叫**哈希函数**一个把查找表中的关键字映射成该关键字对应的地址的函数
- 记为Hash(key)=Addr(数组下标，索引或内存地址等)

### 构造散列表的方法

#### 1.直接定址法

直接取关键字的某个线性函数值为散列地址

#### 2.除留余数法★★★

假定散列表表长为I，选取一个不大于I的最大质数p，利用**取余公式**将关键字映射为散列地址，散列函数为

> [!important] 重点
> H(key)=key％p

**考试题目一般采用的都是除留取余法**，该方法需要注意的是，p不是表长

#### 3.数字分析法

![[72864fb11c3af683bf224479790216f4_MD5.png]]

#### 4.平方取中法

**这种方法取关键字的平方值的中间几位作为散列地址**，具体取多少位根据实际情况而定。

> [!important] 重点
> 这种方法得到的散列地址与关键字的每一位都有关系，因此使得散列地址分布比较均匀，适用于关键字的每位取值都不够均匀或均小于散列函数地址所需的位数。

![[5c0a42b7f429f055d895ec921e1d4e8c_MD5.png]]

****

> [!important] 重点
> 在开放定址法中散列到同一个地址而产生的‘‘堆积问题’’，是**同义词冲突的探查序列和非同义词之间不同的探查序列交织在一起**，导致关键字查询需要经过较长的探测距离，降低了散列的效率。
>
> - ==**堆积现象因冲突而产生，它对存储效率，散列函数和装填因子均不会有影响**==
>
> - **散列表的平均查找长度与装填因子直接相关，表的查找效率并不直接依赖于表中已有表项n或表长m**
>
> ![[7ebe2af903dbb72fdbd809bac1076c78_MD5.png]]
>
> - **表的查找效率取决于散列函数，处理冲突的方法和装填因子。**

### 处理冲突的方法

> [!important] 重点
> **事实上，任何散列函数都不可能绝对避免冲突**

在散列表中插入一个数据元素时，若位置已经被占了，就叫**冲突(碰撞)**

- 减少方法:优化散列函数 
- 处理方法:**拉链法，开放地址法**

#### 拉链法

又叫**链接法（链地址法）**

![[a9dab5d07d4cb71e15612ebd7d42f393_MD5.png]]

**每个单链表由一个或多个同义词构成**，凡是散列地址为i的关键字都可以插入到对应的单链表中（头插法效率最高）

![[f26de65f41c76d94293fbc1a849294c9_MD5.png]]

#### 开放地址法

**开放地址法又叫开放定址法**。当某个关键字key带入哈希函数后，得到的散列地址单元倘若已经存储了其他关键字，**则会使用某种方法为产生冲突的关键字寻址一个新的散列地址**，该散列地址对应的存储区域还未保持其他关键字

<img src="https://fyywww.oss-cn-hangzhou.aliyuncs.com/img_for_typora/202507260725629.png" alt="image-20250726072517183" style="zoom:67%;" />

**开放定址法又分为线性探测法、平方检测法、双散列法、伪随机序列法**

##### 线性探测法

也称**线性探测再散列法**。d~i~=1，2，3，…，m-1.

它的特点是：当冲突发生时，顺序查看表中下一个单元（当探测到表尾时，下一个探测地址回到表首）

> [!important] 重点
> 可能会时第i个散列地址的同义词存入第i+1个散列地址，这样本应存入第i+1个散列地址的元素就争夺第i+2个散列地址的元素的地址…从而造成大量元素在相邻的散列地址上**聚集（或堆积）**起来，大大降低了查找效率

##### 平方探测法

也称**二次探测法**，d~i~=1^2^,-1^2^,2^2^,-2^2^…k^2^,-K^2^(其中k<=m/2，散列表的长度m，必须是一个可以表示称4k+3的素数)

> [!important] 重点
> 平方探测法是一种处理冲突的较好方法，可以避免出现堆积问题。
>
> 
>
> 缺点：**不能探测得到散列表上的所有单元，但至少能探测到一半的单元**

##### 双散列法

> [!important] 重点
> d~i~=i×hash~2~(key)

需要使用两个散列函数。当通过第一个散列函数得到的地址发生冲突时，则利用第二个散列函数计算该关键字的地址增量 

 ==H~i~=(H(key)+i×Hash~2~(key))％m==，初始探测位置H~0~=H(key)%m。i是冲突的次数，初始为0

##### 伪随机序列法

d~i~=伪随机数序列

# 排序

> [!important] 重点
> 内部排序：在排序期间元素全部存放在内存中的排序
>
> 外部排序：在排序期间元素无法全部同时存放在内存中，必须在排序的过程中根据要求不断地在内、外存之间移动的排序

> [!important] 重点
> 算法的稳定性：**经过排序后，能使关键字相同的元素保持原顺序中的相对位置**
>
> ![[ec5b5532612465fcc1f27bf555d1ce28_MD5.png]]

## 内部排序

### 插入排序

#### 直接插入排序★★

思想：每一次将一个**待排序元素**按照其关键字大小**插入已经排好序的一组元素的适当位置上，直到所有元素都被插入**

1. 空间复杂度**O(1)**,仅使用了常数个辅助单元
2. 在排序过程中，向有序子表中逐个地插入元素的操作进行了n-1趟，每趟都分为*比较关键字*和*移动元素*
   1. 最好情况下**O(n)**，原本就是有序的，此时每次插入都只需要比较一次
   2. 最坏情况下**O(n^2^)**，元素顺序是逆序(与排序结果是相反的)，**比较**和**移动**次数都达到了n规模
   3. 平均情况下**O(n^2^**), 
3. 稳定性：**稳定**
4. 适用性：适合顺序存储和链式存储的线性表

#### 折半插入排序 ★★	

[折半插入排序演示](https://www.bilibili.com/video/BV1E1421b7Eb/?spm_id_from=333.337.search-card.all.click&vd_source=523b573b0d781e4f8e447aa67c9fa523)

1. **折半查找只适合顺序表**
2. 一直到low>high才停止折半查找。当mid元素指向当前元素时，仍需要令low=mid+1来保证稳定性。
3. 最终应当将当前元素插入low所指位置(即high+1)
4. 边比较边移动
5. 他与直接插入排序最大的不同就是时间复杂度，约为O(n*log~2~n)，**但是如果数据量大的话，还是O(n^2^)**

#### 希尔排序★★

1. 把待排序表分割成若干特殊子表，**相隔一定增量**的组成一个子表，对各个子表分别进行直接插入排序，每趟处理后都缩小增量，再重复过程；当整个表基本有序时，再对全体进行一次直接插入排序

   > 别搞混淆了，希尔排序相当于第1，3，5，7个组成一组，2，4，6，8组成一组，是分隔一定量的组成一组 

   1. 只可用于**顺序表**
   2. 稳定性：**不稳定**
   3. 空间复杂度 O(1）
   4. 时间复杂度：**未知**，但优于直接插入排序

### 交换排序

#### 冒泡排序

- 从后往前或从前往后**两两比较相邻元素的值**，若为逆序，则交换他们，直到序列比较完。这样过程为一趟冒泡排序，【最多只需要n-1趟排序，最少1趟】
- 每一趟排序都可以使一个元素移动到最终位置，已经确定最终位置的元素后序无需处理
- 如果某一趟排序过程中未出现交换，则算法可以提前结束
- 稳定性:稳定
- 适用性：顺序表链表都可以

#### 快速排序

- 只适合顺序存储的线性表
- 不稳定

- **快速排序是所有内部排序算法中最优的排序算法**

- right从右往左去找比枢轴小的元素去填left的坑

- left从左往右去找比枢轴大的元素去填right的坑
- 初始序列若为顺序或者逆序，则快速排序的平均性能最差(*这样每次选择的元素都是最靠边的元素*)**直接O(n^2^)**==这是个考点==

### 基数排序

思想：**不基于比较和移动进行排序**，而基于关键字各位的大小进行排序

在基数排序中，每个元素在每一轮都会被“移动”一次，移动次数只与“关键字的位数 d”有关，而不是与元素原来的排列顺序有关。 

- ==基数排序与序列的初始状态无关==
- MSD最高位优先   LSD最低位优先
- ==基数排序中建立的队列个数等于进制数==
- 适合场景：①关键字易拆为d且d较小；②每组的关键字的取值范围不大，即r较小；③数据元素个数n较大
- 基数排序需要进行d趟分配和收集操作，一趟分配需要遍历所有关键字O(n)；一趟收集需要合并r个队列O(r);总为O(d(n+r))；
- 稳定性：每趟分配和收集都是从后往前，不会交换相同关键字的相对位置，因此很**稳定**
- 适用性：适合顺序存储和链式存储的线性表
- N个元素进行k路归并的趟数m满足K^m^ =N,即m=log~2~N，向上取整 



### 计数排序（不在考纲内）

但是他的思想多次涉及

计数排序是一种不基于比较的排序算法。他利用数组的下标来确定元素的位置

![[fee4a35a90ad438788612130451a9437_MD5.png]]

计数排序是一种用空间换时间的做法。输出数组的长度为n；辅助的计数数组的长度为k，空间复杂度为O(n+k)。若不把输出数组视为辅助空间，则空间复杂度O(k)

1.当数列最大最小值差距过大时，并不适用于计数排序
比如给定20个随机整数，范围在0到1亿之间，此时如果使用计数排序的话，就需要创建长度为1亿的数组，不但严重浪费了空间，而且时间复杂度也随之升高。

2.当数列元素不是整数时，并不适用于计数排序
如果数列中的元素都是小数，比如3.1415，或是0.00000001这样子，则无法创建对应的统计数组，这样显然无法进行计数排序。

正是由于这两大局限性，才使得计数排序不像快速排序、归并排序那样被人们广泛适用。

### k路平衡归并排序

==目的：减少归并趟数==

需要在内存中分配**k个输入缓冲区**和**1个输出缓冲区**

- 步骤

  - 生成r个初始归并段（对L个记录进行内部排序，组成一个有序的初始归并段）
    - 进行s趟归并排序，S=log~k~r(向上取整)

![[1c3df75485ef1057abbe5058070ad9bf_MD5.png]]

- | 如何进行k路归并                                              |      |      |
  | ------------------------------------------------------------ | ---- | ---- |
  | 1.把k个归并段的块读入k个输入缓冲区                           |      |      |
  | 2.用归并排序的方法从k个归并段中选出几个最小记录暂存在输出缓冲区中 |      |      |
  | 3.当输出缓冲区满时，写出外存                                 |      |      |

  ==外部排序时间开销=读写外存的时间+内部排序的时间+内部归并的时间==，主要时间开销消耗在内外存之间的数据交换上

- 优化思路
  - 增加归并路数k，进行多路平衡归并
    - 代价1：需要增加相应的输入缓冲区
    - 代价2：每次从k个归并段中选出一个最小元素，需要k-1次关键字对比（引入败者树来解决）
  - 减少初始归并段数量r

### 败者树

> [!important] 重点
> 咸鱼认为，如果考题中考到败者树，最有可能的三个出题角度是：
>
> ①问你败者树的作用
>
> ②构造k路归并的败者树时，关键字对比次数是多少？
>
> ③基于构造好的败者树，每次从k个归并段中找到最小关键字，至多/至少需要对比关键字多少次？
>
>  
>
> **下面对着三个问题进行总结**：
>
> ①败者树的作用是：在进行多路归并时，减少关键字对比次数。
>
> ②构造k路归并的败者树时，需要对比关键字 k-1 次。
>
> 例如：构造7路归并的败者树时，需要对比关键字6次。
>
> ③基于构造好的败者树，每次从k个归并段中找到最小关键字 所需的关键字对比次数，取决于败者树的高度和形态。万一考到这类问题，可以先画出败者树的形态，再来分析。
>
> k路归并的败者树，形态上是一棵具有 k 个叶子结点的完全二叉树。
>
> 例如：7路归并的败者树形态如下
>
> ![[099a594f9bb1f8120a125ae44c701cc4_MD5.jpg]] 
>
> 总共有7个叶子结点，每个叶子对应一个归并段。
>
> 基于这棵构造好的败者树，从7个归并段中找到最小关键字，最多需要3次关键字对比；最少需要2次关键字对比。
>
>  
>
> 如果本轮新关键字来自于归并段1，则只需要2次关键字对比即可找到7个归并段中的最小关键字。如下所示：

- 每个初始归并段对应一个叶子结点，把归并段的块数作为叶子的权值
- **归并过程中磁盘I/O次数=归并树的WPL*2**

1. k叉归并树一定是个严格完全k叉树，度只有k和0

2. 构造方式
   1. 补充虚段：设度为0的结点有n~0~个(**初始归并段**)，度为k的结点有n~k~个，归并树的结点总数为n，则有
   
      > 虚段就是**含有0个磁盘块的归并段，不会影响I/O次数**，记作0 
   
      > 1. n=n~k~+n~0~ (总结点数=度为k的结点数+度为0的结点数)
      > 2. n=kn~k~+1 (总结点数=所有结点的度数之和+1)
      > 3. 因此对于严格k叉树有n~0~=(k-1)n~k~+1，由此可得n~k~=(n~0~-1)/(k-1)
      > 4. 若**(n~0~-1)%(k-1)=0**，说明刚好可以构造k叉归并树，==不需要添加虚段==
      > 5. 若**(n~0~-1)%(k-1)=u != 0**，，则需要补充==(k-1)-u==个虚段
